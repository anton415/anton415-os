# Миссия

`anton415-hub` — это приватное веб-приложение и модульный монолит на Go для
одного владельца. Проект объединяет личную продуктивность, финансы, инвестиции
и FIRE-планирование в одну поддерживаемую систему.

## Для кого

- Главный пользователь — владелец проекта.
- Проект также служит учебной площадкой для разработки реального production
  приложения на Go.
- Будущие читатели должны понимать продукт по коду и нескольким коротким
  документам, а не по большой иерархии документации.

## Проблема

Личное планирование обычно разложено по разным инструментам, заметкам,
таблицам и старым репозиториям. Из-за этого сложнее видеть текущее состояние,
безопасно развивать рабочие процессы и использовать AI-assisted development
без устаревшего контекста.

`anton415-hub` решает эту проблему так: важные личные workflows живут в одном
production-приложении с понятными границами модулей.

- Todo: задачи, проекты, даты, приоритеты, ссылки и повторения.
- Finance: месячные факты доходов и расходов.
- Investments: будущий учет портфеля.
- FIRE: будущее долгосрочное планирование.

## Принципы

- Сначала один владелец. Сначала решаем реальный workflow, потом обобщаем.
- Скучный стек. Предпочитать Go, PostgreSQL, TypeScript, Docker и простое
  развертывание лишним движущимся частям.
- Вертикальные срезы. Каждая фича должна включать нужную доменную логику, API,
  хранение, UI и тесты.
- Видимые границы модулей. Todo, Finance, Investments, FIRE, Auth и Platform
  должны оставаться понятными отдельными областями внутри одного репозитория.
- Кодовая база — источник истины. Документация должна кратко описывать текущее
  состояние, а не спорить с ним.
- AI — инженерный помощник. Спеки, фокус ревью, coding agents и автоматизация
  должны делать изменения понятнее и безопаснее.
- Меньше документационного долга. Оставлять только документы, которые помогают
  с продуктовым направлением, локальной разработкой, production-операциями или
  релизными решениями.

## Сейчас не входит в scope

- Командная работа и multi-user collaboration.
- Несколько production-владельцев до появления изоляции данных по пользователю.
- Microservices, Kubernetes, message brokers или отдельные репозитории.
- Широкие финансовые фичи: синхронизация счетов, импорт карт, переводы и
  прогнозирование.
- Импорт/синхронизация инвестиций и FIRE-расчеты, пока ранние модули не станут
  достаточно стабильными.
- AI-фичи внутри продуктового UI без конкретного owner workflow.
- Большая иерархия документации.
